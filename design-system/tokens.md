| XL | 24px | 24pt | 24dp | — |

### Radius guidelines

- Corner radii follow a containment logic: smaller elements inherit tighter radii, larger containers expand outward
- Do not mix different size radii in one screen — use one size to maintain cohesion
- Child elements should not use a larger corner radius than their parent container
- Do not hardcode random radius values (e.g. 10px, 18px) — use defined tokens
- Do not use spacing tokens when applying corner radius
- Avoid modifying each corner individually

### Naming inconsistencies

- Light theme border styles use `Border/` prefix; dark theme uses `Dark/Border/` — consistent
- Dark theme has `Dark/Border/M1 Plus` with no light theme equivalent `Border/M1 Plus` (asymmetric)
- `Inter/PL/400-Underlined` uses "Underlined" (past tense) while `Inter/PM/400-Underline` and `Inter/PS/400-Underline` use "Underline" (present tense)
- Old spacing page uses lowercase tokens (xxxs, xxs, xs, s, m, l, xl, xxl, xxxl); new page uses uppercase (3XS, 2XS, XS, S, M, L, XL, 2XL, 3XL, 4XL, 5XL) — the uppercase ("New") naming is the canonical version
- Old spacing scale has different values at upper end: xl=48px, xxl=56px, xxxl=80px/88px; new scale: XL=40px, 2XL=48px, 3XL=56px, 4XL=64px, 5XL=88px
- `Dark/FG/Success` = #1F7C3E while light `FG/Success` resolves to Green-05 #1F7C3E via style but the semantic desc says it maps to Green-04 in dark

### Tokens used in components but not formally defined here

- Figma variable aliases (VariableID:29701:197XX range) are used extensively in components for color binding but map to external library variables whose names cannot be resolved via this file's API
- Spacing variable aliases (hash-based IDs like `VariableID:af5eea12...`) are used in component padding/gap but reference an external spacing variable collection
- These likely correspond to the spacing and color tokens listed above, but the formal binding is maintained in a separate Figma library file
